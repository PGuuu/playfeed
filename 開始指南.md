# PlayFeed 開始指南（零程式基礎版）

整包東西長這樣：

```
playfeed/
├── index.html    ← 網站本體（feed、按讚、留言、儲存、我的頁面都在這）
├── config.js     ← 後台鑰匙（唯一需要你動手改的檔案，共三個值）
├── setup.sql     ← 資料庫初始化（複製貼上用，不用看懂）
└── games/
    ├── pack1.js  ← 第一批十款遊戲
    ├── pack2.js  ← 像素猜猜
    └── pack3.js  ← 削馬鈴薯
```

運作方式：
- **不登入**：所有遊戲直接玩，往上滑換下一款，完全零門檻
- **要按讚 / 留言 / 儲存收藏時**：才會跳出登入視窗（Phuze，輸入 Email 點信中連結即完成，免密碼）
- **登入後**：右上角變成你的名字，點開就是「我的頁面」——收藏的遊戲和按過讚的遊戲

## 第一步：先讓網站上線（5 分鐘，免費）

1. 打開 https://app.netlify.com/drop
2. 把整個 playfeed 資料夾拖進去
3. 完成。它會給你一個網址（例如 `xxx.netlify.app`），發給朋友就能玩

這時候是「體驗模式」：遊戲全部能玩，但按讚、留言、儲存還不是真的。
**把這個網址記下來，第三步會用到。**

## 第二步：開 Supabase，存互動資料（15 分鐘，免費）

1. 到 https://supabase.com 註冊，按 New project 開一個專案
   （名字隨便取，資料庫密碼記下來，區域選 Northeast Asia (Tokyo)）
2. 左邊選單點 SQL Editor → New query
   → 打開 setup.sql，全選複製、貼上、按 Run
   → 顯示 Success 就完成了資料庫
3. 左邊選單點 Project Settings → API，會看到兩個東西：
   - Project URL（一個網址）
   - anon public key（一串很長的亂碼）
4. 打開 config.js，把這兩個值貼進對應的引號裡，存檔

## 第三步：開 Phuze，讓會員登入變成真的（10 分鐘）

Phuze 是免密碼的 Email 登入服務（就是 Wellnote 用的那套）。
最簡單的做法：**直接跟 Claude 說**——

> 「幫我註冊 PlayFeed 的 Phuze client，網域是 xxx.netlify.app」

Claude 會幫你跑註冊指令，你只要到信箱收 6 位數啟用碼給它，
拿到 `pk_live_` 開頭的 publishable key 之後貼進 config.js 的
`PHUZE_PUBLISHABLE_KEY`，存檔。

兩個要注意的地方：
- Phuze 目前是邀請制 alpha，如果註冊被拒（403），寄信到
  phuze-alpha@edato.me 申請，或用邀請碼
- **網域一定要登記對**：你的 Netlify 網址（例如 `xxx.netlify.app`）
  必須在 Phuze 的授權網域清單裡，不然登入信寄不出去而且不會顯示錯誤。
  在自己電腦上測試（localhost）不用登記，永遠可以用

## 最後：更新網站

config.js 三個值都填好之後，把整個資料夾再拖進 Netlify Drop 一次。
完成——現在讚是真的、留言是真的、收藏會出現在每個人自己的頁面。

## 之後每天上新遊戲

1. 跟 Claude 說：「照 pack1.js 的格式，幫我做這幾款遊戲：（丟你的梗）」
2. 把生成的檔案存成 games/pack4.js（依序編號）
3. 打開 index.html，找到 `<script src="games/pack3.js"></script>`，
   在它下面加一行 `<script src="games/pack4.js"></script>`
4. 資料夾再拖進 Netlify Drop 一次，今天的更新就上線了

## 常見問題

- 登入信沒收到 → 看垃圾信件夾；同一個 Email 一分鐘只能寄一封，別狂按
- 按讚留言後資料沒出現 → 檢查 config.js 三個值是否都填了、
  網域有沒有在 Phuze 登記
- 想刪掉某則留言 → Supabase 後台 → Table Editor → comments → 找到那列刪掉
- 想看有幾個人玩 → Netlify 後台有基本流量數據
- 安全性說明：目前的資料庫政策是「全開」（任何人理論上可以偽造互動），
  對小型社群實驗夠用；如果之後玩的人多了，跟 Claude 說要升級成
  Edge Function 驗證就好
