# PlayFeed 內建遊戲執行規範

這份文件供平台維護者新增內建遊戲包使用。它只規定執行格式與平台邊界，不限制題材、玩法、節奏、難度曲線或創作方向。經典玩法與新玩法都可以，不要把本文件的欄位或程式骨架當成玩法範例。

## 一、輸入邊界

- 垂直滑動保留給 Feed 切換遊戲，不能作為遊戲的必要操作。
- 平台支援點按、按住、放開、左右選擇與水平拖曳。
- 遊戲透過 `input(type, x, y)` 接收 `down`、`move`、`up`、`cancel`。
- `cancel` 只解除按住或拖曳狀態，不可觸發原本屬於 `up` 的發射、計分或結算行為。
- 遊戲不自行監聽 DOM 事件。

## 二、技術格式

內建遊戲存成 `games/packN.js`：

```js
window.GAMES = (window.GAMES || []).concat([
  {
    id: 'unique-id',
    title: '遊戲名稱',
    author: '@作者名',
    tip: '一句話操作說明',
    bg: '#123456',

    create(env) {
      function start() {
        // 重設狀態並開始
      }

      function stop() {
        // 停止動畫與 timer
      }

      function input(type, x, y) {
        // 處理 down / move / up / cancel
      }

      return { start, stop, input };
    }
  }
]);
```

上方只示範結構與生命週期，不代表任何玩法。

執行規則：

- 邏輯畫布固定為 400 × 700，使用 `env.W`、`env.H` 與 `env.ctx`。
- 分數變化呼叫 `env.setScore(score)`；遊戲結束呼叫 `env.over(score)`。
- 音效使用 `env.beep()`，不載入外部音檔。
- 不碰 DOM、不連網、不使用瀏覽器儲存、不載入外部資源。
- `start()` 每次都能完整重設遊戲。
- `stop()` 必須取消 requestAnimationFrame、timer 與其他循環。
- 寫完在 `index.html` 的遊戲包清單加入 `<script src="games/packN.js"></script>`。

## 三、時間與預覽

- 公開投稿目前要求 `duration` 為 20～60 秒；內建遊戲也應填寫合理的預估單局時間。
- 平台會用合法的隨機輸入產生 Feed 預覽。
- 遊戲不必為了預覽改變玩法，但收到隨機輸入時不可卡死、報錯或留下無法停止的循環。

## 四、Remix（選配）

只有在遊戲中確實有適合替換的視覺元素時才加入 `remixSlots`：

```js
remixSlots: [
  { key: 'element', label: '可替換元素', hint: '顯示在遊戲中的元素', default: '●', shape: 'free' }
]
```

繪圖時呼叫 `env.sprite(key, cx, cy, size, flip)`。有素材時平台繪製並回傳 `true`；回傳 `false` 時遊戲再畫原本外觀。

Remix 完全是選配。沒有合適元素時直接省略，不要為了符合格式硬加。

## 五、上架檢查

- Script 能正常載入並註冊遊戲。
- `start()`、`stop()`、`input()` 可重複使用。
- `cancel` 與 `up` 分開處理。
- 所有分數都是有限數字。
- 結束時會呼叫 `env.over()`。
- 滑走後不再繪圖或執行 timer。
- 隨機合法輸入不會造成錯誤。
- 垂直滑動仍能正常切換 Feed。

PlayFeed 只負責提供執行邊界；遊戲應該怎麼設計，由創作者決定。
