# PlayFeed 遊戲規範

之後每一款新遊戲（pack4、pack5…）都必須遵守這份規範。
跟 Claude 說「照遊戲規範.md 做新遊戲」就會照這裡的規則產出。

## 一、互動鐵律（違反就不能上架）

1. **嚴禁上下滑動手勢。**
   上下滑是 feed 的專屬手勢（換上一個／下一個遊戲），遊戲不准搶。
   技術上 `touch-action: pan-y` 會把垂直滑動直接交給捲動——
   就算你寫了垂直手勢，玩家一往下滑畫面就會捲走，遊戲根本收不到。
   - ✅ 允許：點按（tap）、水平拖曳／滑動、按住不放＋放開
   - ❌ 禁止：往上滑、往下滑、畫圈、需要斜向大幅移動的手勢
   - 設計時自問：「玩家會不會下意識做出往上或往下的動作？」會，就改機制。
     （前例：切水果從自由揮砍改成「點一下＝橫斬一刀」；
     削馬鈴薯從往下刷改成「刀子自動巡邏、點按下刀」）

2. **單指操作。** 不用多點觸控、不用重壓、不用陀螺儀。

3. **3 秒看懂。** 不用讀說明就能開始玩；tip 一句話講完操作。

## 二、節奏

- 一局 20～60 秒，死了立刻能重來
- 難度隨時間或關卡遞增（速度變快、目標變小、干擾變多）
- 開局前 5 秒是教學區：節奏放慢，讓玩家用手學會

## 三、實況預覽相容（attract mode）

feed 會在遊戲後面自動試玩當預覽：隨機位置點按 ＋ 偶爾水平滑動。
新遊戲必須在這種隨機輸入下「畫面好看、看得出玩法」：
- 隨機亂點也要有視覺回饋（切到東西、爆分數、失誤震動都可以）
- 玩壞了會自動 game over → 系統自動重開，這是正常循環

## 四、技術格式

存成 `games/packN.js`，內容：

```js
window.GAMES = (window.GAMES || []).concat([
{
  id: '不重複的英文代號', title: '名稱：副標', author: '@作者名', tip: '一句話操作說明', bg: '#背景色',
  create(env) {
    const { ctx, setScore, over } = env;   // 也可用 env.W(400) env.H(700) 和全域 beep()
    // ...
    return {
      start() { /* 重設狀態 + requestAnimationFrame 開跑 */ },
      stop()  { /* 停掉 rAF，設 alive = false */ },
      input(t, x, y) { /* t: 'down' | 'move' | 'up' | 'cancel' */ }
    };
  }
}
]);
```

規則：
- 座標是邏輯畫布 400×700（env.W × env.H），不要用實際像素
- 遊戲結束呼叫 `over(score)`；分數變化呼叫 `setScore(score)`
- 音效用全域 `beep(freq1, freq2, dur, vol, type)`，不要載外部音檔
- 不要碰 DOM、不要載外部資源，全部畫在 ctx 上
- `stop()` 必須真的停乾淨（cancelAnimationFrame + alive flag），
  因為玩家隨時會滑走
- `cancel` 只解除按住／拖曳狀態，不可當成玩家放開而發射或計分
- 寫完在 index.html 的遊戲包清單加一行 `<script src="games/packN.js"></script>`

## 五、Remix（可換圖，建議每個有明確角色/物件的遊戲都加）

想讓玩家上傳自己的圖替換遊戲元素，就在遊戲物件加 `remixSlots`：

```js
remixSlots: [
  { key: 'player', label: '主角', hint: '一句說明', default: '🐔' },
  // ...列出這個遊戲的關鍵可換元素（1～3 個最好）
],
```

畫圖時用引擎提供的 `env.sprite(key, cx, cy, size, flip)`：有換圖就置中等比
畫出並回傳 `true`，沒有就回傳 `false`，讓你畫預設外觀。標準寫法：

```js
const sprite = env.sprite || (() => false);   // create() 開頭，兼容無 sprite 的環境
// ...draw 裡，畫某個物件時：
if (!sprite('player', x, y, 74)) {
  // ...原本用 ctx 畫的預設外觀...
}
```

（若要自己控制繪製，也可用 `env.getSprite(key)` 取原圖，回傳 Image 或 null。
削馬鈴薯就是用 getSprite 把圖裁進橢圓裡。）

系統會自動長出「改造」按鈕、上傳面板、即時預覽。玩家按「儲存成新遊戲」
需登入，之後這個 remix 會存到後端 `remixes` 表，變成 feed 裡一則新遊戲
（排在母遊戲後面，大家都玩得到，也能再被 remix）。上傳圖壓到 240px 存 jsonb。
抽象型遊戲（純幾何、色塊、猜謎）沒有單一可換角色，可不加 slots。
（參考：pack1 的 dodge/boba/mole/slice/sheep、pack4 小雞。）

2. 本機測：確認亂點不會壞、上下滑會正常換遊戲
